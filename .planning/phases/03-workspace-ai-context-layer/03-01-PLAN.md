## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> backend AI config contract | only workspace members with admin rights may edit AI policy |
| backend -> Supabase config rows | workspace verification must remain server-side, not client-asserted |
| Supabase Storage -> knowledge metadata | uploaded files must be referenced safely without exposing shared file paths across workspaces |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | I | workspace AI config rows | mitigate | scope every read/write to the active workspace and preserve RLS on `ai_workspace_configs` |
| T-03-02 | T | uploaded knowledge metadata | mitigate | store only metadata references in config rows and keep file access workspace-scoped in Storage |
| T-03-03 | E | AI config mutation routes | mitigate | verify role permissions server-side before accepting any config or policy change |
</threat_model>
