| Boundary | Description |
|----------|-------------|
| Evolution webhook -> backend ingress | Provider payloads must enter only through the canonical backend route. |
| webhook payload -> workspace data | Payload/channel mapping must not create cross-workspace messages. |
| product reply -> Evolution send | Outbound sends must use backend credentials and never expose provider keys to the browser. |
| validation evidence -> planning docs | Logs must prove behavior without copying secrets or sensitive message content. |
