<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Render env -> Evolution runtime | Runtime configuration must come from Render, not image-baked `.env` defaults. |
| secret value -> repository/docs | Real key values must never cross into tracked files or planning artifacts. |
| backend -> Evolution API | Only backend/server-side paths should hold and use Evolution credentials. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-08-01 | T | Evolution container env loading | mitigate | remove/neutralize internal `.env` override and verify Render env precedence in logs |
| T-08-02 | I | Evolution auth key | mitigate | rotate key out-of-band and scan repo for plaintext occurrences |
| T-08-03 | E | frontend credential exposure | mitigate | verify frontend has no direct Evolution credential path |
</threat_model>
