| browser -> routing endpoints | clients may request classification or apply routing, but backend must enforce workspace and role boundaries |
| routing service -> workspace AI context | routing intelligence must consume workspace-scoped context without leaking or using cross-workspace defaults |
| inbox UI -> supervisor reasoning | reasoning detail must not leak to roles that are not allowed to inspect routing rationale |
